# Sharky Web

Десктопная «YouTube-версия» Sharky: каталог HTML мини-игр поверх того же Supabase, что и мобильная TG-лента ([sharky_play](https://github.com/vgametikok/sharky_play)).

## Страницы

| Файл | Что это |
|---|---|
| `index.html` | Главная: случайная сетка игр + чипы Смесь/Вертикальные/Горизонтальные |
| `search.html` | Поиск + каталог (жанры, ориентация, пагинация) |
| `channel.html?u=<username>` | Канал автора: баннер, подписчики, активные игроки за 28 дней, случайная игра, список игр |
| `game.html?id=<game_id>` | Страница игры: плеер-песочница, лайк/дизлайк/избранное/поделиться, подписка, описание, комментарии |
| `me.html` | Личный кабинет: настройки канала, мои игры, история игр |

## Архитектура

- Статика без сборки: ES-модули (`js/`), один CSS (`css/site.css`). Деплой — GitHub Pages.
- Данные: Supabase (общий проект с мобильной лентой). Чтение страниц — `web_*` SECURITY DEFINER RPC; записи (лайки/подписки/комментарии) — прямые INSERT/DELETE под RLS (`app_uid()`).
- Вход: Google OAuth (Supabase Auth). Первый вход провижнит строку `public.users` (`web_ensure_user`). Гостям доступно всё чтение.
- Игры запускаются в `<iframe sandbox="allow-scripts">` **без** `allow-same-origin`; протокол postMessage (`init/start` → `ready/score/gameover`) совместим с мобильной лентой. Игровое время пишется в `game_stats` (`feed_source='web_game'`).

## Безопасность

- В коде сайта **нет секретов**: только публичный anon-ключ Supabase (данные защищает RLS на сервере).
- CSP на каждой странице; скрипты только со своего origin и cdn.jsdelivr.net; iframe-источники — по белому списку.
- Весь пользовательский контент рендерится через `textContent` (нет innerHTML с данными из БД); цвета/градиенты из БД валидируются перед вставкой в стили.

## Локальный запуск

```
node serve.js 5080
```

(нужен только для предпросмотра; ES-модули не работают с file://)
